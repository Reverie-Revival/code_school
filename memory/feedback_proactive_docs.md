Flag (don't just silently write) documentation needs when: a non-obvious architecture decision gets made (candidate for docs/decisions/), or Supabase schema/RLS changes in a way future-James would need to remember why.

**Why:** Standard Reverie Revival default — keep docs need-driven, not proactively bloated.

**How to apply:** Suggest, don't auto-create, unless James has already asked for the doc.
